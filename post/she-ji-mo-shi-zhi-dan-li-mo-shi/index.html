设计模式之单例模式

2019-06-05

很早项目中就使用过单例模式进行开发，但用的最多的还是命名空间对象来规避重复命名，方便统一管理，其实单例模式还有很多其他的作用，今天就特地学习了一下。

单例模式

概念

单例模式（Singleton）：又被称为单体模式，是只允许实例化一次的对象类, 保证一个类只有一个实例，并提供一个访问它的全局访问点。

作用

单例模式可以让代码更加易懂，可以有效避免定义变量或则方法名时出现重复；通过单例模式来管理代码库的各个模块，使功能模块井井有条；也可以通过单例实现无法修改的静态变量。

通过几个实例来认识单例

通过单例模式实现一个代码库的管理

const JT = {
  Util: {
    util_method () {}
  },
  // 工具模块
  Tool: {
    tool_method () {}
  },
  // ajax 模块
  Ajax: {
      get () {},
      post () {}
  },
  // 其他
  others: {}
}

JT.Util.util_method();

通过单例模式实现无法修改的静态变量

// 无法修改的静态变量
var CONSTANT = (function () {
  // 私有变量
  var constant = {
    MAX_NUM: 100,
    MIN_NUM: 1,
    COUNT: 1000
  };
  return {
    // 取值器
    get (name) {
      return constant[name] ? constant[name] : null;
    }
  }
})();

CONSTANT.get('MAX_NUM'); // 100

惰性单例

延迟创建单例对象。

// 惰性单例
var LazySingle = (function () {
  // 单例实例引用
  var _instance = null;
  // 单例
  function Single() {
    // 这里定义私有属性和方法
    // ...
    return  {
      // 外部可访问的属性和方法
      publicMethod: function () {},
      publicProperty: '1.0'
    }
  }

  // 返回单例
  return function () {
    if (!_instance) return _instance = new Single();
    return _instance;
  }
})()

单例模式的实现通过 JavaScript 本身的一些特性作用域、闭包等来实现的。
